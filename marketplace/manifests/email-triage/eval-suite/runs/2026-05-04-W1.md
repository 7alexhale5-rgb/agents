## What changed in the eval config

Two bugs in the original `promptfooconfig.yaml`:

1. **`expected_category` / `expected_urgency` at top level of golden lines, not inside `vars`.** Promptfoo can only template-substitute (`{{expected_category}}`) from inside `vars`, and JS assertions only see them via `context.vars.X`. Fix: ran a one-pass node script to move both fields into `vars` for all 30 lines. Backup at `golden.jsonl.bak`.
2. **`transformValue: output.category` syntax is non-idiomatic.** Switched to `type: javascript` assertions that access `output` (post-transform) and `context.vars` directly. Cleaner, easier to debug, returns explicit `boolean`.

## What changed in the prompt

v1 → v2 added explicit disambiguation rules across the four boundaries that the baseline failures clustered around:

- **Auto-statements / confirmations / domain-transfer-complete** → deferred (not respond). The action already happened; no reply needed.
- **Pricing changes from services the user actively pays for** → respond (signal the user needs to know about cost shifts).
- **Generic platform notifications** (Spotify Wrapped, Slack notification emails, weekly platform digests, webinar replays) → delete (not unsubscribe). Reserve unsubscribe for content with a real opt-out journey.
- **LinkedIn invitations from a named individual** → deferred (not unsubscribe). Social, not spam.
